# Differentiation - Символьное дифференцирование математических выражений

Проект для символьного дифференцирования математических выражений. Преобразует текстовую запись функции в абстрактное синтаксическое дерево (AST), вычисляет производную и упрощает результат.

## AST модуль

**AST (Abstract Syntax Tree)** — это древовидное представление структуры математического выражения. Каждый узел дерева представляет либо операцию, либо значение, либо переменную.

Пример: `2*x + sin(x)`

### Паттерн Visitor

**Visitor** — поведенческий паттерн, позволяющий добавлять новые операции к структуре данных без изменения самих классов узлов.


##  Парсер

Парсер преобразует текстовую строку с математическим выражением в AST.

### Этапы парсинга
1. Лексический анализ (Tokenizer) — разбивает строку на токены
2. Синтаксический анализ (Parser) — строит AST на основе токенов

Пример:
Входная строка: "sin(x)+2"

Токены:
1. Token(FUNCTION, 'sin', pos=0)
2. Token(LPAREN, '(', pos=3)
3. Token(VARIABLE, 'x', pos=4)
4. Token(RPAREN, ')', pos=5)
5. Token(OPERATOR, '+', pos=6)
6. Token(NUMBER, '2', pos=7)

### Грамматика: 

Expression → Term { ('+' | '-') Term }
Term → Factor { ('*' | '/') Factor }
Factor → Power { Power }          # Неявное умножение
Power → Unary { '^' Unary }       # Правый ассоциативный
Unary → ('+' | '-') Unary | Primary
Primary → NUMBER | VARIABLE | FUNCTION '(' Expression ')' | '(' Expression ')'


## Дифференцирование

Модуль реализует автоматическое дифференцирование с помощью паттерна Visitor.

Упрощение выражений
После дифференцирования результат часто требует упрощения. Simplifier применяет алгебраические преобразования:
-Арифметические упрощения
-Упрощение степеней
-Упрощение знаков
-Группировка множителей

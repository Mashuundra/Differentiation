Вспомогательная утилита для преобразования AST в строку.

Содержит функции для вывода абстрактного синтаксического дерева (AST) в человекочитаемом виде.

Основные функции:
1. to_string(node) — преобразует AST в строку для вывода пользователю
2. to_infix_string(node) — преобразует AST в инфиксную строку с полными скобками (для тестов)

Принцип работы:
Функции рекурсивно обходят AST и формируют строковое представление в зависимости от типа узла.

Числа:      NumberNode(42)    → "42"
            NumberNode(3.14)  → "3.14"
            NumberNode(5.0)   → "5" (целые без .0)

Переменные: VariableNode('x') → "x"

Бинарные операции:
  Сложение: (a + b) → "a + b"
  Вычитание: (a - b) → "a - b"
  Умножение: (a * b) → "a*b"
  Деление: (a / b) → "a/b"
  Степень: (a ^ b) → "a^b"

Функции: FunctionNode('sin', arg) → "sin(arg)"

Унарный минус:
  UnaryOpNode('-', x) → "-x"
  UnaryOpNode('-', (x+1)) → "-(x + 1)" (со скобками)

Расстановка скобок:
Для корректного отображения приоритетов операций функция _needs_parentheses() определяет, нужны ли скобки для операнда:

Правила расстановки скобок:
- Если приоритет операнда ниже приоритета родителя → скобки нужны
- Для правого операнда при вычитании, делении и степени → скобки нужны
- Для левого операнда при степени → скобки нужны

Приоритет операций (от низкого к высокому):
  +, -  → приоритет 1
  *, /  → приоритет 2
  ^     → приоритет 3

Примеры:
  (x + 1) * 2  → "(x + 1)*2"
  (2 * x)^3    → "(2*x)^3"
  x + 1 * 2    → "x + 1*2" (скобки не нужны, так как умножение выше сложения)

Инфиксная строка (to_infix_string):

Используется для тестирования структуры AST.
Добавляет скобки вокруг каждой бинарной операции:
  BinaryOpNode(x, '+', 5) → "(x + 5)"
  BinaryOpNode(2, '*', x) → "(2 * x)"
  BinaryOpNode(2*x, '+', 1) → "((2 * x) + 1)"

Это позволяет точно проверить, правильно ли построено дерево.

Пример полного цикла:
1. Парсер:  "2*x + 1" → AST
2. Дифференцирование: AST → производная (AST)
3. Упрощение: сложный AST → упрощенный AST
4. Вывод: to_string(упрощенный AST) → "2"

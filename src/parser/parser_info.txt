Парсер (синтаксический анализатор) — это программа, которая преобразует текстовую строку
с математическим выражением в структурированное представление (AST).

Парсер состоит из двух основных этапов:
Лексический анализ (Tokenizer) — разбивает строку на токены (слова языка)
Синтаксический анализ (Parser) — строит AST на основе последовательности токенов

exeptions:
Позволяют точно определить тип ошибки
Упрощают обработку ошибок в вызывающем коде
Дают возможность показывать пользователю понятные сообщения

parser:

Пример
Входная строка: "sin(x)+2"
Токены:
1. Token(FUNCTION, 'sin', pos=0)
2. Token(LPAREN, '(', pos=3)
3. Token(VARIABLE, 'x', pos=4)
4. Token(RPAREN, ')', pos=5)
5. Token(OPERATOR, '+', pos=6)
6. Token(NUMBER, '2', pos=7)

Expression → Term { ('+' | '-') Term }
Term → Factor { ('*' | '/') Factor }
Factor → Power { Power }  # Неявное умножение
Power → Unary { '^' Unary }  # Правый ассоциативный
Unary → ('+' | '-') Unary | Primary
Primary → NUMBER | VARIABLE | FUNCTION '(' Expression ')' | '(' Expression ')'


Tokenizer разбивает строку на токены, игнорируя пробелы и обрабатывая числа, переменные, операторы и функции

Parser использует метод рекурсивного спуска для построения AST с учетом:

Приоритета операций
Ассоциативности
Неявного умножения
Вложенных скобок и функций
Система исключений обеспечивает понятную обратную связь при ошибках
